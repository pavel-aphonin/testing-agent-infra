# Спецификация мобильного приложения «Альфа-Мобайл QA»

Документ описывает корректное поведение демо-приложения для целей
автоматического тестирования агентом «Марков». Используется как
эталонный документ для RAG-сверки (`Scenario.rag_document_ids`).

---

## 1. Логин

### Экран `Login`

**Поля формы:**

- `Email` (TextField, `test_id=email-input`) — обязательное поле,
  принимает email по RFC 5322. При пустом значении или некорректном
  формате при попытке входа — отображается сообщение
  «Введите корректный email».
- `Пароль` (SecureTextField, `test_id=password-input`) — обязательное
  поле, минимум 6 символов. При пустом значении — сообщение
  «Введите пароль». При длине менее 6 символов — сообщение
  «Пароль слишком короткий».

**Кнопка `Войти` (`test_id=login-button`):**

- При корректных credentials — переход на экран `Profile`.
- При неверных credentials — сообщение «Неверный email или пароль».
  Это единственное допустимое сообщение для случая ошибки
  аутентификации; формулировки «Неверная карта», «Доступ запрещён»,
  «Ошибка сервера» считаются ошибкой UI.

## 2. Профиль

### Экран `Profile`

Отображается сразу после успешного логина.

**Содержимое:**

- Аватар (первая буква имени).
- Email пользователя (read-only).
- Имя пользователя (TextField, `test_id=name-input`) — редактируемое
  по нажатию кнопки `Редактировать`.
- Телефон (TextField, `test_id=phone-input`) — редактируемое по
  той же кнопке. Допустимый формат `+\d{1,3}\s?\d{3}\s?\d{3}-\d{2}-\d{2}`
  или просто 7-15 цифр со знаком `+` опционально. Невалидный формат
  даёт сообщение «Некорректный номер телефона».

**Кнопки:**

- `Редактировать` (`test_id=edit-button`) — переключает форму в режим
  редактирования.
- `Сохранить` (`test_id=save-button`) — сохраняет изменения. При
  валидной форме показывает alert «Профиль сохранён» и возвращает в
  read-only.
- `Отмена` — возврат без сохранения.
- `Настройки` (`test_id=settings-button`) — переход на экран
  `Settings`.
- `Перевод` (`test_id=transfer-nav-button`) — переход на экран
  `Transfer`.
- `Удалить аккаунт` (`test_id=delete-button`) — открывает confirm:
  «Вы уверены? Это действие нельзя отменить.». При подтверждении —
  переход на `Login`.

## 3. Настройки

### Экран `Settings`

Содержит переключатели тёмной темы и языка. Кнопка `Назад`
возвращает на `Profile`.

## 4. Перевод

### Экран `Transfer`

Открывается с экрана `Profile` по нажатию кнопки «Перевод».

**Поля формы:**

- `Сумма` (TextField, `test_id=amount-input`) — обязательное поле,
  принимает целое положительное число. Единица измерения — рубли.

**Бизнес-правила:**

- При сумме ≤ 0 при попытке перевода — сообщение
  «Введите положительную сумму».
- **При сумме > 100 000 ₽ операция блокируется** с сообщением
  «Превышен лимит». Это единственно допустимая формулировка для
  превышения лимита; сообщения «Неверная карта», «Сумма слишком
  большая», «Ошибка» считаются ошибкой UI и подлежат исправлению.
- При корректной сумме (> 0 и ≤ 100 000) — сообщение
  «Перевод {N} ₽ выполнен» и сброс поля суммы.

**Кнопки:**

- `Перевести` (`test_id=transfer-button`) — выполняет перевод по
  правилам выше.
- `Назад` (`test_id=transfer-back-button`) — возврат на `Profile`.

---

## 5. Общие правила интерфейса

- Все кнопки должны иметь `accessibilityLabel`, совпадающий с
  отображаемым текстом, для совместимости с AXe-инспектором.
- Все поля ввода должны иметь стабильный `test_id` (kebab-case).
- Сообщения об ошибках отображаются под соответствующим полем
  красным цветом (`#e74c3c`).
- Сообщения об успехе — зелёным (`#27ae60`).
