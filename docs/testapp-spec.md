# Спецификация демо-приложения TestApp

## 1. Общее описание

TestApp — мобильное приложение для iOS (React Native), используемое для демонстрации возможностей ИИ-агента тестирования. Приложение имитирует типичное банковское/финтех-приложение с базовыми функциями авторизации, профиля и настроек.

**Платформа:** iOS (iPhone, iPad)
**Фреймворк:** React Native
**Минимальная версия iOS:** 16.0
**Ориентация:** только портретная (iPhone), портретная + ландшафтная (iPad)

---

## 2. Экраны и навигация

### 2.1. Карта экранов

```
Экран входа (Login)
  ├── [Успешный вход] → Профиль (Profile)
  │                       ├── [Настройки] → Настройки (Settings)
  │                       │                   └── [Удалить аккаунт] → Диалог подтверждения
  │                       └── [Редактировать] → Профиль (режим редактирования)
  └── [Ошибка] → Сообщение об ошибке на экране входа
```

### 2.2. Переходы между экранами

| Откуда | Действие | Куда |
|--------|----------|------|
| Login | Успешная авторизация | Profile |
| Login | Неверные данные | Login (с ошибкой) |
| Profile | Нажать «Настройки» | Settings |
| Profile | Нажать «Редактировать» | Profile (режим редактирования) |
| Settings | Нажать «Удалить аккаунт» | Диалог подтверждения удаления |
| Settings | Нажать «Назад» | Profile |
| Profile | Нажать «Выйти» | Login |

## 3. Экран входа (Login)

### 3.1. Элементы интерфейса

| Элемент | Тип | Обязательный | Описание |
|---------|-----|-------------|----------|
| Логотип | Изображение | — | Логотип приложения в верхней части экрана |
| Заголовок | Текст | — | «Вход в аккаунт» |
| Поле Email | TextField | Да | Ввод email-адреса пользователя |
| Поле Пароль | SecureTextField | Да | Ввод пароля, символы скрыты |
| Кнопка «Войти» | Button | — | Отправка формы авторизации |
| Ссылка «Забыли пароль?» | Link | — | Переход на экран восстановления (не реализован в демо) |

### 3.2. Валидация полей

**Email:**
- Не может быть пустым → сообщение: «Введите email»
- Должен содержать символ @ и точку → сообщение: «Некорректный формат email»
- Максимальная длина: 255 символов

**Пароль:**
- Не может быть пустым → сообщение: «Введите пароль»
- Минимальная длина: 6 символов → сообщение: «Пароль должен содержать минимум 6 символов»
- Максимальная длина: 128 символов

### 3.3. Тестовые учётные данные

| Email | Пароль | Результат |
|-------|--------|----------|
| test@example.com | password123 | Успешный вход |
| admin@test.com | admin | Ошибка: пароль слишком короткий |
| invalid | любой | Ошибка: некорректный email |
| test@example.com | wrongpass | Ошибка: «Неверный email или пароль» |

### 3.4. Поведение при авторизации

1. Пользователь вводит email и пароль
2. Нажимает кнопку «Войти»
3. Кнопка переходит в состояние загрузки (индикатор вращения, кнопка недоступна)
4. При успехе — переход на экран профиля
5. При ошибке — красное сообщение под формой, кнопка снова доступна
6. iOS может показать системный диалог «Сохранить пароль?» — это стандартное поведение iOS, не элемент приложения

### 3.5. Системные диалоги

После успешного входа iOS может показать диалог:
- **«Save Password?»** (или «Сохранить пароль?» для русской локали)
- Кнопки: «Save Password» / «Not Now»
- Это системный диалог iOS, не контролируется приложением
- Агент должен уметь закрыть его, нажав «Not Now»

## 4. Экран профиля (Profile)

### 4.1. Элементы интерфейса

| Элемент | Тип | Описание |
|---------|-----|----------|
| Аватар | Изображение | Круглое фото пользователя (или заглушка с инициалами) |
| Имя пользователя | Текст | Полное имя: «Иван Петров» |
| Email | Текст | Email пользователя: test@example.com |
| Телефон | Текст | Номер телефона: +7 (999) 123-45-67 |
| Кнопка «Редактировать» | Button | Переключение в режим редактирования |
| Кнопка «Настройки» | Button | Переход на экран настроек |
| Кнопка «Выйти» | Button | Выход из аккаунта, возврат на экран входа |

### 4.2. Режим просмотра

- Все поля отображаются как текст (нередактируемые)
- Доступны кнопки «Редактировать», «Настройки», «Выйти»

### 4.3. Режим редактирования

При нажатии «Редактировать»:
- Имя, телефон становятся текстовыми полями (TextField)
- Email остаётся нередактируемым (серый текст)
- Кнопка «Редактировать» заменяется на «Сохранить» и «Отмена»
- При сохранении — данные обновляются, возврат в режим просмотра
- При отмене — изменения отбрасываются

### 4.4. Валидация при редактировании

**Имя:**
- Не может быть пустым → сообщение: «Введите имя»
- Минимальная длина: 2 символа
- Максимальная длина: 100 символов
- Допустимые символы: буквы, пробелы, дефис

**Телефон:**
- Формат: +7 (XXX) XXX-XX-XX
- Маска ввода применяется автоматически
- Должен содержать 11 цифр

| Язык | Selector | Русский | Выбор языка: Русский / English |

### 5.2. Поведение переключателей

- Переключатели сохраняют состояние немедленно (без кнопки «Сохранить»)

- Тёмная тема меняет цвета приложения в реальном времени
- Смена языка перезагружает все тексты на экране

## 6. Удаление аккаунта

### 6.1. Диалог подтверждения

При нажатии «Удалить аккаунт» появляется модальный диалог:

- **Заголовок:** «Удалить аккаунт?»
- **Текст:** «Это действие нельзя отменить. Все ваши данные будут удалены безвозвратно.»
- **Кнопка «Удалить»:** красная, подтверждает удаление
- **Кнопка «Отмена»:** серая, закрывает диалог

### 6.2. Поведение после подтверждения

1. Показывается индикатор загрузки
2. Аккаунт удаляется (в демо — имитация)
3. Пользователь перенаправляется на экран входа
4. Показывается сообщение: «Аккаунт успешно удалён»

## 7. Нефункциональные требования

### 7.1. Производительность

- Время загрузки экрана: не более 500 мс
- Время отклика на нажатие: не более 100 мс
- Анимации переходов: 300 мс

### 7.2. Доступность (Accessibility)

- Все интерактивные элементы имеют accessibility labels
- Минимальный размер кнопки: 44x44 pt (Apple HIG)
- Контрастность текста: соответствует WCAG AA (4.5:1)
- Поддержка VoiceOver для всех экранов
- Все изображения имеют альтернативный текст

### 7.3. Обработка ошибок

- Ошибки сети: «Нет подключения к интернету. Проверьте соединение и попробуйте снова.»
- Таймаут запроса (>10 сек): «Сервер не отвечает. Попробуйте позже.»
- Неизвестная ошибка: «Что-то пошло не так. Попробуйте позже.»

### 7.4. Безопасность

- Пароль не отображается в открытом виде (SecureTextField)
- Сессия истекает через 30 минут неактивности
- Данные авторизации хранятся в iOS Keychain
- Логи не содержат паролей и токенов

---

## 8. Известные ограничения демо-версии

1. Ссылка «Забыли пароль?» не ведёт никуда (placeholder)
2. Данные профиля не сохраняются между запусками
3. Push-уведомления не отправляются реально
4. Биометрия не работает в iOS Simulator
5. Удаление аккаунта — имитация (данные не удаляются)
6. Только один тестовый пользователь (test@example.com / password123)
